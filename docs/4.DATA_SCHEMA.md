# Data Schema (V2)

This is the current storage model used by `pystocks/`.

## 1) High-Level Layout

- `data/ib_products.csv`: ETF instrument universe (symbol/exchange/conid/isin/currency)
- `data/auth_state.json`: Playwright browser state for IBKR session cookies
- `data/pystocks.db`: scraper registry + HTTP call logs (not fundamentals payload storage)
- `data/fundamentals/`: canonical fundamentals PIT storage
- `data/research/`: telemetry and research outputs

## 2) SQLite Registry (`data/pystocks.db`)

Table `instruments`:

- `conid` (PK), `symbol`, `exchange`, `isin`
- `last_scraped_fundamentals`
- `last_status_fundamentals`

Table `scraper_logs`:

- per-request logs: `conid`, `timestamp`, `endpoint`, `status_code`, `error_message`

Use case:

- scheduling/skip logic (`already scraped today`)
- operational trace of endpoint status behavior

## 3) Fundamentals Canonical Storage (`data/fundamentals/`)

### 3.1 CAS blobs (`data/fundamentals/blobs/`)

- full raw endpoint payloads, zstd-compressed JSON
- content-addressed by `sha256`
- file sharding pattern: `blobs/<hash[0:2]>/<hash[2:4]>/<hash>.json.zst`

### 3.2 Event index (`data/fundamentals/events.db`)

SQLite manifest with:

- conid, endpoint, endpoint_slug
- `observed_at`, `effective_at`, `effective_source`
- payload hash/blob path/raw+compressed sizes
- optional `source_file`

Dedupe key:

- `UNIQUE(conid, endpoint, effective_at, payload_hash)`

Analytics-row manifest table:

- `analytics_rows` stores normalized dataset lineage and dedupe metadata.
- key columns: `analytics_name`, `row_key`, `endpoint_event_id`, `conid`, `effective_at`, `payload_hash`, `parquet_path`.
- dedupe key: `UNIQUE(analytics_name, row_key, payload_hash)`

### 3.3 Endpoint-parquet events (`data/fundamentals/parquet/`)

Partitioning:

- `endpoint=<slug>/year=<YYYY>/month=<MM>/*.parquet`

Row shape:

- fixed metadata columns (`conid`, endpoint fields, hash, sizes, timestamps)
- dynamic scalar fields for analytics
  - most endpoints: flattened payload scalar columns with prefix `f__...`
  - complex endpoints (`dividends`, `ownership`): canonical analytics columns with prefix `a__...`
- list helper fields (`f__...__len`, and some `f__...__json` for scalar lists)

Schema evolution:

- new payload fields appear as new columns
- old rows simply have `NULL` for unseen columns

### 3.4 Normalized analytics parquet datasets (`data/fundamentals/parquet/`)

Partitioning:

- `analytics=<name>/year=<YYYY>/month=<MM>/*.parquet`

Current datasets:

- `analytics=dividends_events`
- `analytics=dividends_industry_metrics`
- `analytics=ownership_trade_log`

Row shape:

- base lineage columns: `analytics_name`, `row_key`, `endpoint_event_id`, `endpoint`, `conid`, `observed_at`, `effective_at`, `payload_hash`
- dataset-specific normalized fields (e.g., `event_date`, `amount`, `trade_date`, `action`, `metric_id`, etc.)
- all rows remain traceable back to the raw CAS payload hash

### 3.5 DuckDB query layer (`data/fundamentals/fundamentals.duckdb`)

Contains views over parquet partitions:

- `endpoint_events_all`
- `endpoint_<slug>` per endpoint
- `endpoint_catalog`
- `analytics_rows_all`
- `analytics_<name>` per analytics dataset
- `analytics_catalog`

Important:

- this DuckDB file is a query surface; parquet files remain the underlying event store

## 4) Effective Date Policy

Endpoint events are assigned effective dates independently of scrape date:

1. endpoint-native date for discrete endpoints
   - default: `as_of_date`/`asOfDate`
   - exception: `morningstar` prefers max `publish_date` found anywhere in the payload when present, then falls back to `as_of_date`
2. fallback to `ratios.as_of_date`
3. fallback to modal (most common) discrete endpoint date from same snapshot
4. fallback to observed scrape date

This supports irregular discrete time series by endpoint.

## 5) Complex Endpoint Rules (`dividends`, `ownership`)

Storage remains two-layer and consistent:

1. raw full payload in CAS blobs
2. analytics parquet representation for querying

Specific handling:

- `dividends`: normalized snapshot + `dividends_events` + `dividends_industry_metrics`; embedded price series is excluded from normalized analytics datasets.
- `dividends`: numeric fields are normalized to numbers in analytics parquet (including percent strings converted to numeric fractions, e.g. `1.09% -> 0.0109`), while original display strings remain in `formatted_value` where applicable.
- `ownership`: normalized snapshot + `ownership_trade_log`; embedded price series is excluded from normalized analytics datasets; `NO CHANGE` rows are excluded from trade-log analytics.

## 6) Research and Telemetry Outputs (`data/research/`)

Research policy calibration outputs:

- `research_yields.csv`
- `research_correlations_summary.csv`

Fundamentals run telemetry outputs:

- `fundamentals_run_telemetry_<timestamp>.json`
- `fundamentals_run_telemetry_latest.json`

Optional compare output:

- `fundamentals_telemetry_compare_<timestamp>.csv`

## 7) Legacy Paths

- `data/preprocessed/` is legacy from older preprocessing flow and is not part of V2 fundamentals storage.
- `src/` notebooks/scripts still contain older `ib_async` analysis workflows; those are separate from current `pystocks/` ingestion runtime.
