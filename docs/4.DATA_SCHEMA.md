# Data Schema (Current)

This schema describes the current normalized storage/query model in `pystocks/`.

## 1) Directory Layout

- `data/pystocks.db`
  - instrument registry + scrape logs
- `data/fundamentals/blobs/`
  - CAS raw endpoint payloads (zstd JSON)
- `data/fundamentals/events.db`
  - manifest tables: `blobs`, `endpoint_events`
- `data/fundamentals/parquet/endpoint=.../year=.../month=...`
  - slim endpoint snapshot parquet rows
- `data/factors/ibkr_factor_features/endpoint=.../conid=...`
  - long factor-feature parquet rows
- `data/prices/ibkr_mf_performance_chart/conid=...`
  - price chart series parquet rows
- `data/sentiment/ibkr_sma_search/conid=...`
  - sentiment search series parquet rows
- `data/ownership/ibkr_ownership_trade_log/conid=...`
  - ownership trade-log series parquet rows
- `data/dividends/ibkr_dividends_events/conid=...`
  - dividends events series parquet rows
- `data/fundamentals/fundamentals.duckdb`
  - DuckDB views over parquet stores

## 2) Manifest DB (`events.db`)

### `blobs`

- `hash` (PK)
- `blob_path`
- `raw_size_bytes`
- `compressed_size_bytes`
- `created_at`

### `endpoint_events`

- `id` (PK)
- `conid`
- `endpoint`
- `endpoint_slug`
- `observed_at`
- `effective_at`
- `effective_source`
- `payload_hash`
- `blob_path`
- `payload_size_raw`
- `payload_size_compressed`
- `source_file`
- `inserted_at`

Dedupe constraint:

- `UNIQUE(conid, endpoint, effective_at, payload_hash)`

## 3) Effective Date Rules

Applied in `fundamentals_store.py`:

1. Endpoint-own date where available (`as_of_date` / `asOfDate`), with endpoint-specific extraction.
2. `morningstar` special case: use latest `publish_date` found anywhere in payload.
3. Fallback to `ratios` effective date.
4. Fallback to modal discrete endpoint date in the same snapshot.
5. Final fallback to observed scrape date.

`effective_source` records which rule was applied.

## 4) Parquet Grains

### Endpoint snapshots

Path:
- `data/fundamentals/parquet/endpoint=<slug>/year=<YYYY>/month=<MM>/*.parquet`

Grain:
- One row per persisted endpoint event.

Purpose:
- Lineage and audit surface (not the primary factor-analysis table).

### Factor features

Path:
- `data/factors/ibkr_factor_features/endpoint=<slug>/conid=<conid>/*.parquet`

Grain:
- One numeric feature per row per endpoint event.

Key fields:
- `endpoint_event_id`, `endpoint`, `conid`, `observed_at`, `effective_at`, `payload_hash`
- `feature_name`, `feature_group`, `feature_value`, `feature_source`
- `row_key`, `inserted_at`, `source_file`

### Dedicated series

All series share lineage columns (`endpoint_event_id`, `conid`, `effective_at`, `payload_hash`, etc.) and endpoint-specific value fields.

- `price_chart_series_all`: `trade_date`, `price`, `open`, `high`, `low`, `close`, `debug_y`, ...
- `sentiment_search_series_all`: `trade_date`, sentiment metrics (no stripped OHLC/price keys), ...
- `ownership_trade_log_series_all`: `trade_date`, `action`, `shares`, `value`, `holding`, `party`, ...
- `dividends_events_series_all`: `event_date`, `amount`, `currency`, `description`, `record_date`, `payment_date`, ...
