# Data Schema (Current)

This document describes the active storage/query model and the tail pipeline artifacts.

## 1) Directory Layout
- `data/fundamentals/blobs/`
  - CAS raw endpoint payloads (zstd JSON)
- `data/fundamentals/events.db`
  - manifest tables: `blobs`, `endpoint_events`
- `data/fundamentals/parquet/endpoint=.../year=.../month=...`
  - slim endpoint snapshot parquet rows
- `data/factors/ibkr_factor_features/endpoint=.../conid=...`
  - long factor-feature parquet rows
- `data/prices/ibkr_mf_performance_chart/conid=...`
  - raw price chart series parquet rows
- `data/prices/ibkr_mf_performance_chart_clean/conid=...`
  - cleaned price series parquet rows
- `data/sentiment/ibkr_sma_search/conid=...`
  - sentiment search series parquet rows
- `data/ownership/ibkr_ownership_trade_log/conid=...`
  - ownership trade-log series parquet rows
- `data/dividends/ibkr_dividends_events/conid=...`
  - dividends event series parquet rows
- `data/research/`
  - telemetry, quality reports, analysis outputs
- `data/fundamentals/fundamentals.duckdb`
  - DuckDB views over parquet stores + operational tables (`instruments`, `scraper_logs`)

## 2) Manifest DB (`events.db`)
### `blobs`
- `hash` (PK)
- `blob_path`
- `raw_size_bytes`
- `compressed_size_bytes`
- `created_at`

### `endpoint_events`
- `id` (PK)
- `conid`
- `endpoint`
- `endpoint_slug`
- `observed_at`
- `effective_at`
- `effective_source`
- `payload_hash`
- `blob_path`
- `payload_size_raw`
- `payload_size_compressed`
- `source_file`
- `inserted_at`

Dedupe constraint:
- `UNIQUE(conid, endpoint, effective_at, payload_hash)`

## 3) Effective Date Rules
Applied in `pystocks/fundamentals_store.py`:
1. Endpoint-own date (`as_of_date` / `asOfDate`) when present.
2. `morningstar`: latest nested `publish_date`.
3. Fallback to `ratios` effective date.
4. Fallback to modal discrete endpoint date in snapshot.
5. Final fallback to observed scrape date.
