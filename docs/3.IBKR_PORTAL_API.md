# IBKR Portal API Reference (Project Usage)

This document maps IBKR endpoints to current project usage and practical behavior observed in production runs.

## Base Paths

- Fundamentals: `https://www.interactivebrokers.ie/tws.proxy/fundamentals/`
- Morningstar: `https://www.interactivebrokers.ie/tws.proxy/mstar/`
- Impact: `https://www.interactivebrokers.ie/tws.proxy/impact/`
- Sentiment: `https://www.interactivebrokers.ie/tws.proxy/sma/`

## Core Endpoints

### `landing/{conid}?widgets=...`

Role:
- Primary low-cost completeness probe.
- Source of teaser presence booleans used for dynamic routing.

### `mf_profile_and_fees/{conid}`

Role:
- Always fetch.
- Stable high-value metadata (objective, fees, reports, profile).

### `mf_holdings/{conid}`

Role:
- Always fetch.
- High-yield and foundational for sector/country exposure factors.

### `mf_ratios_fundamentals/{conid}`

Role:
- Conditional fetch (typically when objective/top10/ratios teaser indicates likely signal).
- High value for valuation and z-score factors.

### `mf_lip_ratings/{conid}`

Role:
- Conditional fetch (best predictor is landing overall ratings teaser).

### `dividends/{conid}`

Role:
- Always fetch.
- Returns either rich history or no-dividend summary shape.

### `mstar/fund/detail?conid={conid}`

Role:
- Currently fetched unconditionally due strong yield and low penalty.

## Historical Endpoints with Fallback

### `mf_performance/{conid}?risk_period={P}&statistic_period={P}`

Current strategy:
- Try `P in [10Y, 5Y, 3Y, 1Y, 6M]` until useful payload is found.
- Save selected period in snapshot (`performance_period`).

### `mf_risks_stats/{conid}?period={P}`

Current strategy:
- Try `P in [10Y, 5Y, 3Y, 1Y, 6M]` until useful payload is found.
- Save selected period in snapshot (`risk_stats_period`).

## Low-Yield / Optional Endpoints

### `impact/esg/{conid}`

Observed behavior in current universe:
- Very frequent `400` responses.
- Near-zero useful payload yield.

Current policy:
- Disabled by default in fundamentals pipeline.

Note:
- Landing `mf_esg` widget already provides high-level ESG values for many products.

### `ownership/{conid}`

Observed behavior:
- Frequently `404`.
- Useful payload is rare.

Current policy:
- Disabled by default; opt-in only for targeted studies.

### `sma/request?type=tick&conid={conid}`

Observed behavior:
- Mostly `404`, very low yield.

Current policy:
- Disabled by default.

### `sma/request?type=high_low&conid={conid}`

Observed behavior:
- Low yield (better than tick, still limited).

Current policy:
- Not part of default fundamentals snapshot.

### `sma/request?type=search&conid={conid}&from=...&to=...&bar_size=1D&tz=-60`

Observed behavior:
- Better yield than tick/high_low in current sample.

Current policy:
- Included conditionally (broader gate than ownership-only).

## Status Codes to Treat as Normal in This Domain

- `200`: payload may still be sparse or structurally variant.
- `404`: common for unsupported tabs on specific instruments.
- `400`: common for some optional/supplementary endpoints (notably `impact/esg`).
- `401/403`: auth/session expiration; should trigger reauth workflow.
