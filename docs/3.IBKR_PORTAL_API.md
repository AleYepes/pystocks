# IBKR Portal API Reference (Current Project Usage)

This document reflects what `pystocks/` currently calls and how calls are gated.

## Base Prefixes Used

- Fundamentals: `https://www.interactivebrokers.ie/tws.proxy/fundamentals/`
- Morningstar: `https://www.interactivebrokers.ie/tws.proxy/mstar/`
- Sentiment: `https://www.interactivebrokers.ie/tws.proxy/sma/`
- Impact: `https://www.interactivebrokers.ie/tws.proxy/impact/`

## Auth Check Endpoints Used by `session.py`

- `/tws.proxy/portfolio/accounts`
- `/tws.proxy/iserver/auth/status`
- `/tws.proxy/fundamentals/landing/756733?widgets=objective`

A stored browser state is treated as valid only if at least one auth check returns a non-empty successful payload.

## Fundamentals Endpoints Used by `fundamentals.py`

### Probe endpoint

- `landing/{conid}?widgets=...`
  - always called first for baseline context

### Always-on endpoints

- `mf_profile_and_fees/{conid}?sustainability=UK&lang=en`
- `mf_holdings/{conid}`
- `mf_ratios_fundamentals/{conid}`
- `mf_lip_ratings/{conid}`
- `dividends/{conid}` (short and long payload families are both retained)
- `mstar/fund/detail?conid={conid}`
- `mf_performance_chart/{conid}?chart_period=MAX`
- `sma/request?type=search&conid={conid}&from=...&to=...&bar_size=1D&tz=-60`
- `ownership/{conid}`
- `impact/esg/{conid}?accounts=<ACCOUNT_ID>`
  - `accounts` is derived from `data/auth_state.json` portal cookie paths (`portfolio2`/`acesws` account segment).

### Period-fallback endpoints

- `mf_performance/{conid}?risk_period={P}&statistic_period={P}`
- fallback order: `10Y -> 5Y -> 3Y -> 1Y -> 6M`

## Complex Endpoint Analytics Policy

`pystocks/` keeps a consistent two-layer storage policy for all endpoint payloads:

1. raw wire response in CAS blobs (`data/fundamentals/blobs/...`)
2. analytics parquet datasets (`data/fundamentals/parquet/...`)

For complex endpoints:

- `dividends`
  - CAS keeps the full raw payload (including embedded price series when present).
  - analytics parquet writes canonical snapshot fields + normalized `dividends_events` and `dividends_industry_metrics` datasets.
  - embedded price series from dividend history is excluded from normalized analytics rows.
- `ownership`
  - CAS keeps the full raw payload (including `ownership_history` price series).
  - analytics parquet writes canonical snapshot fields + normalized `ownership_trade_log`.
  - `NO CHANGE` actions are excluded from `ownership_trade_log` analytics rows.

## Deferred Families

Short-selling chart families remain deferred in this phase (not fetched/stored by fundamentals runtime):

- `hmds/lastLine`
- `hmds/studyLine?source=FeeRate`
- `hmds/studyLine?source=Inventory`

Sentiment snapshot families remain excluded (not fetched/stored by fundamentals runtime):

- `sma/request?type=tick`
- `sma/request?type=high_low`

`sentiment_search` storage rule:

- price/OHLC fields are stripped before storage (`high`, `low`, `price`, `price_change_p`, `close`, `open`, `price_change`).

## Status Codes in This Domain

- `200`: success; payload may still be sparse
- `400`: common for unsupported optional endpoints (notably ESG)
- `404`: common for endpoint/instrument combinations not supported by IBKR
- `401/403`: auth expiry; triggers reauth flow in `fundamentals.py`

## Product Universe Endpoint (Separate from Fundamentals)

Used by `product_scraper.py`:

- `POST https://www.interactivebrokers.ie/webrest/search/products-by-filters`
- pulls ETF universe into `data/ib_products.csv`

## Price-Series API Status

Current runtime now ingests IBKR web price series via:

- `mf_performance_chart/{conid}?chart_period=MAX`

Storage policy for this family is separate from fundamentals endpoint parquet partitions (see schema doc).

Additional price sources are still candidates for future validation/backup.

Suggested exploration order:

1. Inspect portal network calls for chart/history widgets for a known ETF conid.
2. Test candidate historical-bar endpoints under `tws.proxy` with existing auth cookies.
3. Validate response schema stability across conids/time ranges.
4. Choose one canonical price endpoint and implement in `pystocks/` with PIT-compatible storage.

Potential candidate family to evaluate first:

- `tws.proxy/iserver/marketdata/...` historical bar endpoints (to be verified in-project).
