# IBKR Portal API: Project Endpoint Policy

This is the live endpoint policy used by `pystocks/fundamentals.py`.

## Base Proxy Prefixes

- `fundamentals/*` under `/tws.proxy/fundamentals/`
- `mstar/*` under `/tws.proxy/mstar/`
- `sma/*` under `/tws.proxy/sma/`
- `impact/*` under `/tws.proxy/impact/`

## Session/Auth Checks

`session.py` validates auth with authenticated API calls:

- `/tws.proxy/portfolio/accounts`
- `/tws.proxy/iserver/auth/status`
- `/tws.proxy/fundamentals/landing/756733?widgets=objective`

## Fetched Endpoint Set

`fundamentals.py` always attempts the same endpoint set per conid.

- `landing/{conid}?widgets=...`
- `mf_profile_and_fees/{conid}?sustainability=UK&lang=en`
- `mf_holdings/{conid}`
- `mf_ratios_fundamentals/{conid}`
- `mf_lip_ratings/{conid}`
- `dividends/{conid}`
- `mstar/fund/detail?conid={conid}`
- `mf_performance_chart/{conid}?chart_period=MAX`
- `sma/request?type=search&conid={conid}&from=1995-01-01 00:00&to=<now>&bar_size=1D&tz=-60`
- `ownership/{conid}`
- `impact/esg/{conid}?accounts=<account_id>` (falls back to no account param if account ID is unavailable)
- `mf_performance/{conid}?risk_period={P}&statistic_period={P}` with fallback order:
  - `10Y -> 5Y -> 3Y -> 1Y -> 6M`

## Explicit Exclusions

The following families are intentionally not fetched in default runtime:

- `mf_risks_stats`
- `sma/request?type=tick`
- `sma/request?type=high_low`
- `hmds/lastLine`
- `hmds/studyLine?source=FeeRate`
- `hmds/studyLine?source=Inventory`

## Endpoint-Specific Storage Rules

- `price_chart`:
  - Raw payload in fundamentals CAS.
  - Daily bar series in `data/prices/ibkr_mf_performance_chart/`.

- `sentiment_search`:
  - Raw payload is sanitized before persistence to remove:
    - `high`, `low`, `price`, `price_change_p`, `close`, `open`, `price_change`
  - Sentiment series rows stored in `data/sentiment/ibkr_sma_search/`.

- `ownership`:
  - Raw payload retained in CAS.
  - Trade-log rows stored in `data/ownership/ibkr_ownership_trade_log/`.
  - `NO CHANGE` actions excluded.

- `dividends`:
  - Raw payload retained in CAS.
  - Dividend event rows stored in `data/dividends/ibkr_dividends_events/`.

## Status-Code Expectations

- `200`: success (payload may still be sparse)
- `400`/`404`: common for unsupported endpoint/instrument combinations (especially optional ESG)
- `401`/`403`: auth expired, triggers reauth flow
