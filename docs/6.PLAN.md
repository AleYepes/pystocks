## Work Package 3: Add Price Preprocessing Module
1. Create `/pystocks/price_preprocess.py`.
2. Add `PricePreprocessConfig` with defaults:
   `min_history_days=252`, `max_missing_ratio=0.30`, `max_internal_gap_days=20`, `stale_run_max_days=5`, `outlier_z_threshold=50.0`.
3. Implement deterministic pipeline:
   - Load from `price_chart_series_all`.
   - Deduplicate per `(conid, trade_date)` using latest `(effective_at, observed_at, endpoint_event_id)`.
   - Choose `close_price = COALESCE(close, price)`.
   - Flag invalid rows: non-positive price, invalid OHLC relationships when OHLC present.
   - Compute returns with no interpolation by default.
   - Compute stale-run flags and outlier flags.
   - Compute per-conid coverage and eligibility.
4. Persist outputs:
   - `/data/prices/ibkr_mf_performance_chart_clean/conid=*/*.parquet`.
   - `/data/research/price_quality_report_latest.json`.
5. Register views in `/pystocks/fundamentals_store.py` refresh flow:
   - `price_chart_series_clean_all`.
   - `returns_daily_clean`.
   - `price_quality_catalog`.

### WP3 Exit Criteria
1. Clean parquet store exists and is queryable.
2. Quality report exists and includes eligibility reasons.
3. New views are present in DuckDB after refresh.
