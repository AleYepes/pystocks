# Agent Catch-Up Guide (Current State)

This file replaces older planning notes with the current implementation snapshot.

## 1) What exists now
- Fundamentals ingestion and normalization are active in `pystocks/`.
- Tail pipeline exists and runs through:
  1. `python -m pystocks.cli preprocess_prices`
  2. `python -m pystocks.cli run_analysis`
  3. or both via `python -m pystocks.cli run_pipeline`

## 2) Canonical runtime modules
- `pystocks/session.py`
- `pystocks/product_scraper.py`
- `pystocks/fundamentals.py`
- `pystocks/fundamentals_store.py`
- `pystocks/price_preprocess.py`
- `pystocks/analysis.py`
- `pystocks/cli.py`

## 3) Known design constraints
- Analysis is intentionally lean for now.
- Available factors depend on feature coverage in `factor_panel_long_daily`.
- If factor features do not include value/growth or small/large tags, only market factor may be produced.

## 4) Validation baseline for new agents
Run:
```bash
python -m pystocks.cli refresh_fundamentals_views
python -m pystocks.cli preprocess_prices
python -m pystocks.cli run_analysis
python -m pytest -q pystocks/tests
```

Check:
- `data/research/price_quality_report_latest.json`
- `data/research/analysis_summary_latest.json`

## 5) Legacy context
- `src/` notebooks/scripts are retained as historical context, not the primary runtime path.
- Old plan artifacts are superseded by this document.
