# Fundamentals Refactor Summary (V2)

## Purpose

The pipeline moved from daily monolithic JSON snapshots to a **point-in-time endpoint event model** that is easier to query, deduplicate, and extend.

## Before vs After

Before:

- V1 per-conid/day JSON snapshots in `data/fundamentals/<conid>/<date>.json`
- weak dedupe semantics
- difficult schema evolution for analysis

After (current):

- landing-first, policy-gated endpoint fetch in `pystocks/fundamentals.py`
- endpoint events persisted by `pystocks/fundamentals_store.py`
- CAS compressed raw blobs + SQLite manifest + endpoint-partitioned Parquet + DuckDB views
- run telemetry emitted every scrape run

## Current Scrape Policy

Always fetched:

- `mf_profile_and_fees/{conid}?sustainability=UK&lang=en`
- `mf_holdings/{conid}`
- `mstar/fund/detail?conid={conid}`

Conditionally fetched (landing-driven):

- `mf_ratios_fundamentals/{conid}` when `has_top10 or has_objective`
- `mf_lip_ratings/{conid}` when `has_overall_ratings`
- `dividends/{conid}` when landing indicates dividends content
- `sma/request?type=search...` when `has_overall_ratings`
- `mf_performance/{conid}?risk_period=P&statistic_period=P` when `has_perf or has_objective`
- `mf_risks_stats/{conid}?period=P` when `has_risk or has_objective`

Low-yield opt-in (`FETCH_LOW_YIELD_ENDPOINTS=True`):

- `impact/esg/{conid}`
- `ownership/{conid}` (gated by ownership teaser)
- `sma/request?type=tick&conid={conid}` (gated by ownership teaser)

Period fallback order for performance/risk:

- `10Y -> 5Y -> 3Y -> 1Y -> 6M`

## Current Storage Model

- CAS blobs (`sha256`, zstd JSON): `data/fundamentals/blobs/`
- Event index + dedupe manifest: `data/fundamentals/events.db`
  - dedupe key: `(conid, endpoint, effective_at, payload_hash)`
- Endpoint parquet partitions: `data/fundamentals/parquet/endpoint=<slug>/year=<YYYY>/month=<MM>/...`
- DuckDB query views: `data/fundamentals/fundamentals.duckdb`
  - `endpoint_events_all`
  - `endpoint_<slug>`
  - `endpoint_catalog`

Effective date policy is endpoint-aware and irregular by design:

1. endpoint `as_of_date`/`asOfDate` when present for discrete endpoints
2. fallback to `ratios.as_of_date`
3. fallback to modal (most common) discrete as-of in same snapshot
4. fallback to observed scrape date

## Auth and Reliability Status

- `session.py` now validates auth state using authenticated API checks (not UI-only heuristics).
- `fundamentals.py` retries via interactive `reauthenticate()` when 401/403 occurs.
- Manual login is still required when cookies expire.

## Outputs to Inspect After Any Run

- `data/research/fundamentals_run_telemetry_latest.json`
- `data/fundamentals/events.db`
- `data/fundamentals/parquet/endpoint=...`
- `data/fundamentals/fundamentals.duckdb`
