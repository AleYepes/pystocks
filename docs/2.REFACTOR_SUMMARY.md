# Runtime Runbook (Agent Onboarding)

This is the fastest way for an uninformed agent to run and validate the current repo.

## 1) Prerequisites
- Python environment with dependencies installed.
- Valid IBKR session flow through `pystocks/session.py`.
- Instrument registry populated in DuckDB (create/update via `python -m pystocks.cli scrape_products`).

## 2) Standard Execution Flow
1. Validate/login session:
- `python -m pystocks.session`

2. Refresh ETF universe (if needed):
- `python -m pystocks.cli scrape_products`

3. Scrape fundamentals/series payloads:
- `python -m pystocks.cli scrape_fundamentals --limit 100`

4. Optional full normalized rebuild from CAS events:
- `python -m pystocks.cli backfill_fundamentals_normalized`

5. Refresh DuckDB views:
- `python -m pystocks.cli refresh_fundamentals_views`

## 3) Tail-End Pipeline
1. Preprocess prices:
- `python -m pystocks.cli preprocess_prices`

2. Run analysis:
- `python -m pystocks.cli run_analysis`

3. One-command tail run:
- `python -m pystocks.cli run_pipeline`

## 4) Immediate Outputs To Check
- Telemetry: `data/research/fundamentals_run_telemetry_latest.json`
- Manifest DB: `data/fundamentals/events.db`
- Query DB: `data/fundamentals/fundamentals.duckdb`
- Price quality report: `data/research/price_quality_report_latest.json`
- Analysis summary: `data/research/analysis_summary_latest.json`

## 5) Fast Validation Queries (DuckDB)
```sql
SELECT * FROM endpoint_catalog ORDER BY endpoint;
SELECT * FROM factor_features_catalog ORDER BY feature_group, feature_name LIMIT 200;
SELECT * FROM price_chart_series_catalog ORDER BY conid LIMIT 50;
SELECT * FROM price_quality_catalog ORDER BY conid LIMIT 50;
SELECT COUNT(*) AS n_returns FROM returns_daily_clean;
```

## 6) Operational Notes
- Price date parsing prioritizes `timestamp_ms` (`x`) and only falls back to `debugY`.
- `sentiment_search` strips embedded OHLC/price fields before persistence.
- `ownership_trade_log` excludes `NO CHANGE` rows.
- Analysis is intentionally lean; factor availability depends on feature coverage in `factor_panel_long_daily`.
