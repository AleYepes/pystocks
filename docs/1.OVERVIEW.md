# Project Overview: PyStocks

PyStocks is a quantitative investment pipeline for ETF universe management, fundamental data collection, factor modeling, and portfolio analysis.

## Current State (Feb 21, 2026)

The project has moved from OCR-heavy TWS scraping to IBKR Portal JSON ingestion.

- Core fundamentals scraping is implemented in `pystocks/fundamentals.py`.
- Session/auth handling is managed by `pystocks/session.py`.
- Instrument metadata and scrape logs are stored in SQLite (`data/pystocks.db`).
- Point-in-time endpoint payloads are stored in a CAS + Parquet event model under `data/fundamentals/`:
  - compressed raw blobs (`data/fundamentals/blobs/`)
  - endpoint-partitioned Parquet events (`data/fundamentals/parquet/endpoint=...`)
  - event index (`data/fundamentals/events.db`)
  - DuckDB query layer (`data/fundamentals/fundamentals.duckdb`)

## Strategic Direction

1. Keep the pipeline resilient to heterogeneous instrument schemas.
2. Minimize unnecessary API calls to reduce load and rate-limit risk.
3. Preserve PIT integrity for downstream regression/backtest workflows.
4. Build robust flattening/ETL from variable nested JSON to analysis tables.

## What Changed Recently

- Period fallback was added for performance/risk endpoints (`10Y -> 5Y -> 3Y -> 1Y -> 6M`).
- Landing-first dynamic endpoint routing was introduced in fundamentals.
- Endpoint/teaser correlation research was run at scale (`sample_size=500`) to inform hardcoded fetch logic.
