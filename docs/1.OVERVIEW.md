# PyStocks: Start Here

This repository is an IBKR-first ETF ingestion and analysis pipeline with a dual-layer storage model and a lean tail-end analysis pass.

## In-Scope Runtime Components
- Session/auth: `pystocks/session.py`
- Product universe scrape: `pystocks/product_scraper.py`
- Fundamentals + series scrape: `pystocks/fundamentals.py`
- CAS + parquet + DuckDB materialization: `pystocks/fundamentals_store.py`
- Price preprocessing and eligibility: `pystocks/price_preprocess.py`
- Daily factor analysis: `pystocks/analysis.py`

## Runtime Model
1. Scrape endpoint payloads per conid.
2. Persist raw payloads to CAS blobs (`data/fundamentals/blobs/`).
3. Persist normalized parquet stores (endpoint snapshots, factor features, dedicated series stores).
4. Refresh DuckDB views in `data/fundamentals/fundamentals.duckdb`.
5. Run tail pipeline:
   - `preprocess_prices`
   - `run_analysis`

## Important Context
- `src/` contains historical notebooks/scripts and is not the primary runtime path.
- Current production code is under `pystocks/`.
