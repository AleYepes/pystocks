# PyStocks: Start Here

This repo is an IBKR-first data ingestion pipeline for ETF fundamentals and related time series, built for point-in-time factor analysis.

## What Is In Scope Right Now

- Authenticated scraping through `pystocks/session.py`.
- ETF universe discovery through `pystocks/product_scraper.py`.
- Fundamentals ingestion through `pystocks/fundamentals.py`.
- Dual-layer storage and query views through `pystocks/fundamentals_store.py`.

## Current Storage Model

Every fetched endpoint follows a dual-layer model:

1. Raw layer: endpoint payload as persisted in CAS blobs (`data/fundamentals/blobs`).
2. Analytics layer: parquet datasets for:
- slim endpoint snapshot rows (`data/fundamentals/parquet/endpoint=...`).
- long factor-feature rows (`data/factors/ibkr_factor_features/endpoint=.../conid=...`).
- dedicated series stores:
  - `data/prices/ibkr_mf_performance_chart/`
  - `data/sentiment/ibkr_sma_search/`
  - `data/ownership/ibkr_ownership_trade_log/`
  - `data/dividends/ibkr_dividends_events/`

DuckDB views are materialized in `data/fundamentals/fundamentals.duckdb`.

Note: `sentiment_search` removes embedded price/OHLC fields before CAS and parquet persistence.

## Endpoint Coverage (Default Runtime)

`fundamentals.py` fetches:

- `landing`
- `profile_and_fees`
- `holdings`
- `ratios`
- `lipper_ratings`
- `dividends`
- `morningstar`
- `price_chart`
- `performance` (period fallback)
- `sentiment_search`
- `ownership`
- `esg` (with account query param when available)

Not fetched by default:

- `risk_stats`
- sentiment snapshots (`tick`, `high_low`)
- short-selling families (`hmds/lastLine`, `hmds/studyLine`)
