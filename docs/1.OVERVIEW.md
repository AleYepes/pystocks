# Project Overview: PyStocks

PyStocks is a quantitative investment pipeline to optimize a portfolio of ETFs. It handles fundamental data collection, historic price series collection, factor modeling, multi-factor regression, and efficient frontier portfolio analysis and optimization.

## Current State (Feb 22, 2026)

The new version 2.0 of PyStocks drafted in `pystocks/` is currently centered on **ETF fundamentals ingestion from IBKR Portal** with point-in-time storage.

Implemented core:

- `pystocks/session.py`: browser-assisted auth with API-backed state validation.
- `pystocks/product_scraper.py`: ETF universe discovery into `data/ib_products.csv`.
- `pystocks/fundamentals.py`: landing-first fundamentals scraping with endpoint gating, period fallback, telemetry, and reauth handoff.
- `pystocks/fundamentals_store.py`: V2 storage backend (CAS blobs + event index + endpoint-partitioned Parquet + DuckDB views).

Current canonical fundamentals storage is under `data/fundamentals/`:

- raw payload blobs: `data/fundamentals/blobs/`
- event manifest: `data/fundamentals/events.db`
- analytics parquet partitions: `data/fundamentals/parquet/endpoint=...`
- SQL query layer: `data/fundamentals/fundamentals.duckdb`

Supporting runtime metadata:

- `data/pystocks.db`: instrument registry and scraper HTTP call logs.
- `data/research/fundamentals_run_telemetry_*.json`: run-level endpoint/call/useful/status telemetry.

## What Is Not Yet Implemented

- A production-grade price-series ingestion pipeline in `pystocks/` for the full ETF universe.
- Automated completeness-gated handoff from fundamentals coverage to price-series collection.
