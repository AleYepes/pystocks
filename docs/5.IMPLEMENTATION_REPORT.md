# Operational Checklist (Replaces Old Point-in-Time Report)

This document is intentionally stable. It avoids stale one-off counts and focuses on repeatable validation.

## 1) Minimum Post-Run Checks

1. Telemetry file exists:
- `data/research/fundamentals_run_telemetry_latest.json`

2. Endpoint events exist:
```sql
SELECT COUNT(*) AS n_events FROM endpoint_events_all;
```

3. Factor features exist:
```sql
SELECT COUNT(*) AS n_factor_rows FROM factor_features_all;
```

4. Series stores are visible:
```sql
SELECT COUNT(*) AS n_price_rows FROM price_chart_series_all;
SELECT COUNT(*) AS n_sentiment_rows FROM sentiment_search_series_all;
SELECT COUNT(*) AS n_ownership_rows FROM ownership_trade_log_series_all;
SELECT COUNT(*) AS n_dividend_rows FROM dividends_events_series_all;
```

## 2) Data Integrity Checks

- `NO CHANGE` ownership rows should not exist:
```sql
SELECT COUNT(*) AS no_change_rows
FROM ownership_trade_log_series_all
WHERE upper(action) = 'NO CHANGE';
```

- Factor lineage should join back to endpoint events:
```sql
SELECT
  COUNT(*) AS factor_rows,
  SUM(CASE WHEN e.event_id IS NULL THEN 1 ELSE 0 END) AS missing_event_join,
  SUM(CASE WHEN e.payload_hash IS DISTINCT FROM f.payload_hash THEN 1 ELSE 0 END) AS payload_hash_mismatch
FROM factor_features_all f
LEFT JOIN endpoint_events_all e
  ON f.endpoint_event_id = e.event_id;
```

- Sentiment stripped-price fields should not appear in schema:
```sql
SELECT column_name
FROM information_schema.columns
WHERE table_name = 'sentiment_search_series_all'
  AND column_name IN ('price','open','high','low','close','price_change','price_change_p');
```
