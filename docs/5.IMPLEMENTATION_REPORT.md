# Implementation Report (Status as of Feb 22, 2026)

## 1) Scope of Completed Work

Completed in `pystocks/`:

- Session/auth hardening in `pystocks/session.py`
  - auth state is validated via API calls, not only page heuristics
  - stale cookie states no longer get treated as valid
  - interactive reauth path is used by fundamentals on 401/403
- Fundamentals policy and execution in `pystocks/fundamentals.py`
  - landing-first endpoint routing
  - period fallback for `mf_performance` and `mf_risks_stats`
  - endpoint-level telemetry for call counts/useful payloads/status mix
- Storage migration to V2 in `pystocks/fundamentals_store.py`
  - CAS compressed blobs
  - endpoint event index with dedupe constraints
  - endpoint-partitioned parquet events
  - DuckDB query views

Removed/retired from canonical runtime path:

- legacy monolithic fundamentals JSON snapshots
- legacy migration helper script usage as a normal runtime step
- discovery capture corpus as a required source of truth

## 2) Current Runtime Pipeline

Normal run sequence:

1. `python -m pystocks.session`
   - ensures valid auth cookies in `data/auth_state.json`
2. `python -m pystocks.fundamentals --force=True --limit=<N>`
   - scrapes conids from `data/ib_products.csv`
   - persists endpoint events into V2 storage
   - refreshes DuckDB views at run end
   - writes run telemetry JSON

No separate ETL or store command is required after a standard fundamentals run.
